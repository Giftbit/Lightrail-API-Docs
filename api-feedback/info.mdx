#### Parameters

| Name | Located in | Description | Required | Schema |
| ---- | ---------- | ----------- | -------- | ------ |
| Idempotency-Key | header | Client-generated key to ensure idempotent request processing.  Prevents duplicate operations when the same request is made multiple times. Include this header for all state-changing operations to ensure safe retry behavior and prevent unintended duplicates.  | No | string |
| account_id | path | Unique identifier for an account within your organization's hierarchy.  Account IDs are used consistently across all API operations to reference specific accounts in role assignments, control configurations, and resource management.  | Yes | [UUID](#uuid) |
| role_name | path | Name of the role to operate on within the account.  Role names identify specific permission containers and must match existing roles configured for the account. Use the List Account Roles endpoint to discover available role names.  | Yes | [AccountRole](#accountrole) |
| actor_id | path | Unique identifier for an actor (user or API client) within your organization.  Actor IDs represent both human users and programmatic API clients that can be assigned to roles. Use the List Role Members endpoint to discover actor IDs assigned to specific roles.  | Yes | string |
